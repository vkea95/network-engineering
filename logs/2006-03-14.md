- 3.Day 2 tasks

你看到：

< HTTP/2 301

这是一个 非常正常且重要的 HTTP 现象。这正是 Day 2 要理解的内容之一。

⸻

1 301 是什么

HTTP 状态码 301 的含义是：

301 = Moved Permanently

意思：

这个 URL 被永久重定向到另一个地址

服务器在告诉客户端：

请去访问另一个 URL。

⸻

2 为什么会发生 301

最常见的原因有三个：

① HTTP → HTTPS

例如：

http://example.com

服务器会返回：

301 → https://example.com

强制使用 HTTPS。

⸻

② www 重定向

例如：

example.com

重定向到：

www.example.com


⸻

③ canonical URL

网站统一 URL。

例如：

/old-page

重定向到：

/new-page


⸻

3 查看重定向位置

执行：

curl -I https://google.com

你可能会看到：

HTTP/2 301
location: https://www.google.com/

关键字段：

location

表示 新的地址。

⸻

4 让 curl 自动跟随重定向

默认 curl 不会自动跳转。

使用：

curl -L https://google.com

参数：

-L = follow redirect

你就会得到最终页面。

⸻

5 观察完整重定向过程（很重要）

执行：

curl -v https://google.com

你可能会看到：

< HTTP/2 301
< location: https://www.google.com/

* Issue another request
> GET / HTTP/2

这表示：

client
↓
request google.com
↓
server 返回 301
↓
client 再请求新地址


⸻

6 常见 HTTP 状态码

你今天只需要记这几个：

状态码	含义
200	成功
301	永久重定向
302	临时重定向
403	禁止访问
404	页面不存在
500	服务器错误


⸻

7 一个小实验（推荐）

执行：

curl -I http://google.com

你几乎一定会看到：

301

因为 Google 强制 HTTPS。

9 一个工程师的小技巧

如果你想 只看状态码：

curl -o /dev/null -s -w "%{http_code}" https://google.com

输出：

200

这个在 脚本自动化 中非常常见。
